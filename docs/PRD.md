# TranscriptPro — YouTube 长视频转录桌面客户端

> 长视频转录，本地运行，完整逐字稿

## 1. 产品概述

### 1.1 一句话描述

一款付费桌面应用，支持 YouTube 长视频本地转录，输出带时间戳的完整逐字稿，支持多语言翻译和多格式导出。

### 1.2 为什么做这个

- YouTubeTranscriptFree（现有免费网站）已有稳定流量
- 70+ 用户主动发邮件要求支持长视频转录
- 免费版依赖 YouTube 自带字幕，无字幕或长视频时无法工作
- 市场上没有好用的、面向普通用户的 YouTube 长视频转录桌面工具

### 1.3 产品定位

```
YouTubeTranscriptFree（免费网站）
  │  获客入口，处理短视频/有字幕的视频
  │
  ▼  "需要长视频转录？试试 Pro 版"
TranscriptPro（付费客户端）
     长视频、无字幕视频、本地运行、隐私安全
```

### 1.4 产品名称（待定）

- TranscriptPro
- TranscriptTube Pro
- YouTubeTranscript Pro
- SubScribe（双关：字幕 + 订阅）

暂用 **TranscriptPro** 作为工作名。

## 3. 核心功能

### 3.1 功能架构

```
TranscriptPro
├── 视频输入
│   ├── YouTube URL（主要）

├── 输出处理
│   ├── 完整逐字稿（带时间戳）
│   ├── AI 翻译（可选，多语言）

│   └── 多格式导出（SRT/VTT/TXT/PDF/Markdown）
│
└── 用户体验
    ├── 任务队列（支持批量）
    ├── 历史记录
    ├── 全文搜索
    └── 一键复制/导出
```

## 5. 设计规范：品牌与 UI 一致性

### 5.1 统一品牌

客户端与 YouTubeTranscriptFree 网站保持统一品牌形象：

| 维度 | 策略 |
|------|------|
| **Logo** | 使用与网站相同的 Logo，客户端版本加 "Pro" 标识 |
| **产品名** | 网站: TranscriptTube / 客户端: TranscriptTube Pro（或最终确定的名字） |
| **配色** | 与网站主色调一致，深浅模式同步 |
| **字体** | 统一字体（如 Inter + 思源黑体），中英文一致 |

### 5.2 UI 风格对齐

用户从网站导流到客户端，视觉体验不能断裂：

- **输入区**：URL 输入框样式、位置与网站一致
- **结果展示**：转录文本排版、时间戳样式与网站对齐
- **按钮/操作**：复制、导出等操作按钮风格统一
- **圆角/间距/阴影**：保持相同的设计语言

### 5.3 差异化（客户端独有）

在统一基础上，客户端增加桌面端特有元素：
- 侧边栏：历史记录列表
- 拖拽区域：本地文件导入
- 系统托盘：后台转录通知
- 原生菜单栏：文件/编辑/帮助

### 6.2 转录策略（四层降级）

```
输入 YouTube URL
      │
      ▼
第一层: 尝试提取 YouTube 自带字幕
      │
    有字幕? ──Yes──→ 直接输出（秒级完成，零成本）
      │
     No
      │
      ▼
第二层: yt-dlp 下载音频流 → Fast-Whisper 本地转录
      │
      │  注: YouTube 视频/音频分开存储(DASH格式)，
      │  yt-dlp 直接下载音频流，不需要下载完整视频
      │  音频体积: ~5-20MB/小时（vs 视频 500MB-2GB）
      │
    下载成功? ──Yes──→ Whisper 转录 → 输出
      │
     No（YouTube 反爬/地区限制/yt-dlp 版本过旧）
      │
      ▼
第三层: 用户手动上传
      │  提示: "自动下载失败，请手动下载视频/音频后拖入"
      │  用户可用浏览器插件或其他方式获取文件
      │
      ▼
    Whisper 本地转录 → 输出
      │
    硬件太差/转录失败?
      │
      ▼
第四层: Groq API 云端转录（V2，需用户自带 Key）
      │
      ▼
    输出
```

**为什么字幕优先？**
- YouTube 自带字幕质量通常比 Whisper 好（人工或官方 ASR）
- 零成本，秒级返回
- 只有没字幕的视频才需要 Whisper
- 免费网站已经验证了这个策略

**音频下载成功率：** ~95%+ 公开视频可直接下载。yt-dlp 是 GitHub 90K+ stars 的项目，社区活跃，YouTube 反爬更新后通常 1-7 天内出补丁。年龄限制/地区限制视频需要 cookie 或代理，极少数情况走第三层手动上传兜底。

### 6.3 模型下载与离线能力

#### 首次设置下载内容

用户选择 Whisper 模型后，下载的是语音识别模型权重文件：

| 模型 | 下载大小 | 下载内容 | 转录质量 |
|------|---------|---------|---------|
| tiny | ~75 MB | model.bin + config + tokenizer | 基础，适合速度优先 |
| base | ~140 MB | 同上 | 够用 |
| **small（推荐）** | **~460 MB** | 同上 | 质量好，速度快 |
| medium | ~1.5 GB | 同上 | 很好 |
| large | ~3 GB | 同上 | 最佳 |

存储位置：`~/.transcriptpro/models/whisper-{size}/`
模型只需下载一次，后续使用不再重复下载。

#### 离线能力

| 操作 | 需要网络？ | 原因 |
|------|-----------|------|
| YouTube URL → 提取字幕 | ✅ 需要 | 访问 YouTube 服务器 |
| YouTube URL → 下载音频 | ✅ 需要 | 从 YouTube 下载音频流 |
| **本地文件 → Whisper 转录** | **❌ 不需要** | **模型在本地，纯 CPU/GPU 推理** |
| AI 翻译 | ✅ 需要 | 调用 LLM API |
| 导出 SRT/TXT/Markdown | ❌ 不需要 | 本地文件操作 |
| 查看/搜索历史记录 | ❌ 不需要 | SQLite 本地存储 |
| License 验证 | ⚠️ 首次 + 每 72h | 验证后缓存，断网期间可用 |

**核心卖点：模型下载完成后，本地文件转录完全离线可用。**
飞机上、无网络环境下，拖入本地视频 → 转录 → 导出，全程不联网。

### 6.4 长视频处理策略

2 小时视频 = ~120 分钟音频，直接喂给 Whisper 会 OOM 或极慢。

```
分段策略:
1. ffmpeg 按 10 分钟切分音频（有 30 秒重叠）
2. 每段独立送 Whisper 转录
3. 合并结果，重叠部分去重对齐
4. 输出连续的带时间戳文本

2 小时视频 → 12 个 10 分钟片段 → 逐段转录 → 合并
```

进度可以精确到段：`已完成 7/12 段，预计剩余 3 分钟`

### 7.2 转录性能估算

以 **1 小时视频** 为基准，使用 small 模型：

| 硬件 | 转录时间 | 体验 |
|------|---------|------|
| Mac M1/M2 16GB | ~6-10 分钟 | ✅ 流畅 |
| Mac M1 8GB | ~10-15 分钟 | ✅ 可用 |

## 8. 付费方案

### 8.1 定价策略

| | 免费体验 | 标准版 $19.99 | 专业版 $39.99 |
|---|---------|-------------|-------------|
| **视频时长上限** | 30 分钟/次 | 60 分钟/次 | 无限制 |
| **每月次数** | 3 次 | 100 次/月 | 无限制 |
| **Whisper 模型** | 全部 | 全部 | 全部 |
| **导出格式** | SRT/VTT/TXT/Markdown | SRT/VTT/TXT/Markdown | SRT/VTT/TXT/Markdown |
| **AI 翻译** | ❌ | ❌ | ✅ |
| **AI 总结** | ❌ | ❌ | ✅ |

设计原则：
- **不限制基础体验**：模型和导出格式全版本开放，不在核心转录功能上卡用户
- **靠时长和次数区分层级**：标准版 60 分钟覆盖大多数视频，超长视频推动升级专业版
- **AI 功能做专业版差异化**：翻译和总结是高价值增值功能，需要用户自带 LLM API Key

### 8.2 为什么一次性买断

- 目标用户讨厌订阅制（"又一个月费"）
- 本地运行，没有持续的服务器成本（除 AI 翻译用户自带 Key）
- 一次性付费降低决策门槛
- 买断 + 免费更新 1 年，之后大版本升级可选付费

### 8.3 成本结构

```
用户付费: $19.99 - $39.99

你的成本:
├── 开发成本：一次性投入（自己干）
├── 服务器成本：$0（本地运行，无服务器）
├── License 分发：LemonSqueezy 抽成 5% + $0.50/笔
├── Apple 开发者账号：$99/年（macOS 签名）
├── 代码签名证书（Windows）：$0-200/年
└── 域名/官网：~$100/年

每单净利润:
├── $19.99 × 95% - $0.50 = ~$18.50
└── $39.99 × 95% - $0.50 = ~$37.50
```

**纯利润率 ~92%。** 没有 API 成本、没有服务器成本。

## 14. 运维与更新机制

### 14.1 yt-dlp 自动更新（MVP 必须）

yt-dlp 和 YouTube 是猫鼠游戏，版本过旧会导致下载失败。

```
策略:
├── App 启动时后台检查 yt-dlp 最新版本
├── 有新版 → 静默更新（pip install -U yt-dlp）
├── 下载失败时 → 提示"正在更新下载引擎..." → 更新后自动重试
└── yt-dlp 作为独立组件管理，不硬编码进安装包
```

### 14.2 App 自动更新

```
MVP 方案:
├── 用 GitHub Releases 托管安装包
├── Tauri 内置 updater 检查新版本
├── 启动时检查 → 提示用户下载更新（不强制）
└── 后期可做静默后台更新
```

### 14.3 临时文件清理

转录过程产生的临时音频文件（~5-20MB/小时视频），需自动管理：

```
策略:
├── 转录完成后自动删除临时音频文件
├── 设置页提供"清理缓存"按钮（手动清理）
└── 只保留转录文本结果，不保留音频/视频
```

### 14.4 语言选择

Whisper 自动检测对混合语言（如中英夹杂技术视频）表现不佳：

```
策略:
├── 默认"自动检测"
├── 提供手动指定语言下拉选项
├── 转录结果页提示："如果转录有误，请尝试手动指定语言后重新转录"
└── 支持语言列表：英/中/日/韩/法/德/西/葡/俄 等 Whisper 支持的 99 种语言
```
